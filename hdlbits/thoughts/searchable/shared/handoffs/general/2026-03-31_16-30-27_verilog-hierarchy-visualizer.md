## Learnings

### Architecture Decisions (Locked In)
1. **No Yosys WASM** — 47MB unpacked is overkill. We only need hierarchy extraction (module defs, ports, instantiations), not synthesis/optimization.
2. **Custom Verilog parser in JS** (~400 LOC, <5KB) — Key insight: at module scope, `identifier identifier (` is *always* a module instantiation. No ambiguity with function calls. Handles ~90%+ of real RTL.
3. **ELK.js for layout** (~550KB gzip) — Only heavy dependency. Unavoidable for professional orthogonal routing + port-aware layout. Can lazy-load via Web Worker.
4. **Vanilla SVG rendering** (~300 LOC) — Inspired by HDElk but without SVG.js dependency (saves 66KB). Use `document.createElementNS` directly.
5. **Prism.js textarea overlay** (~12KB gzip) for editor — No CodeMirror 6 (would add ~80-200KB). Note: `@codemirror/lang-verilog` doesn't exist; would need legacy mode adapter.
6. **Vanilla SVG pan-zoom** (69 LOC) — viewBox-based, no library needed.
7. **Vite project** for DX — still deploys as static site.

### Key Research Findings
- **tree-sitter-verilog WASM**: ~6-8MB total (web-tree-sitter 4.5MB + grammar ~1-3MB). Parser.c is ~45MB of generated C. Too heavy for this use case.
- **ELK.js cannot be tree-shaken** — GWT compiles entire Java codebase into one blob. [Issue #6 (2017)](https://github.com/OpenKieler/elkjs/issues/6) still open.
- **dagre** (32KB gzip) has NO port support and NO orthogonal routing — not viable for EDA diagrams.
- **netlistsvg** has NO hierarchy support — [Issues #37, #64](https://github.com/nturley/netlistsvg/issues/37) requested but never implemented.
- **HDElk** has the best JSON API for HDL diagrams (`inPorts`, `outPorts`, `edges`) but no npm package, no ESM, last updated 2021.
- **Custom Sugiyama layout** would be ~1000+ LOC for orthogonal port-to-port routing — not worth it when ELK.js exists.

### Parser Edge Cases (V1 Decisions)
| Pattern | V1 Action |
|---------|-----------|
| `generate if/for` blocks | Flatten — extract all instantiations |
| `` `ifdef `` guards | Include all branches |
| Escaped identifiers `\name` | Skip (~5% of RTL) |
| Parameterized widths | Show as expression string |
| Packages/imports | Ignore |
| Non-ANSI module headers | Warn + skip port extraction |

### Bundle Size Comparison (from research)
| Component | Our Choice | Size (gzip) |
|-----------|-----------|-------------|
| Verilog parser | Custom JS tokenizer | ~5KB |
| Layout engine | ELK.js (layered + orthogonal) | ~550KB |
| SVG rendering | Vanilla DOM | ~10KB |
| Editor | Prism.js + textarea overlay | ~12KB |
| Pan-zoom | Vanilla viewBox | ~1.5KB |
| **Total** | | **~580KB** |
